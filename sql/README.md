# 💾 SQL

This folder contains all SQL scripts used in the **Formula 1 Driver Performance Intelligence** project. The scripts follow a structured analytics workflow, starting from data validation and progressing to advanced analytical queries, reusable database views, and business insights.

---

## 📂 SQL Workflow

| File | Description |
|------|-------------|
| **01_Data_Cleaning.sql** | Performs data validation, checks for duplicates, missing values, blank records, and verifies dataset quality before analysis. |
| **02_Exploratory_Analysis.sql** | Answers key business questions through descriptive statistics and exploratory SQL queries. |
| **03_Advanced_SQL.sql** | Implements advanced SQL techniques including CTEs, Window Functions, Ranking Functions, CASE statements, and Subqueries. |
| **04_Views.sql** | Creates reusable SQL Views for reporting and frequently used analyses. |
| **05_Business_Insights.sql** | Translates analytical findings into meaningful business insights and recommendations. |

## 📊 Business Questions Answered

The SQL analysis explores questions such as:

- Which drivers have achieved the highest race wins?
- Which drivers have the highest career efficiency?
- Which countries have produced the most successful Formula 1 drivers?
- Which decade produced the strongest drivers?
- How do active and retired drivers compare?
- Which drivers consistently convert race entries into victories?
- What factors contribute to championship success?

---

## 🎯 Objective

The objective of this SQL analysis is to transform raw Formula 1 driver statistics into meaningful insights by applying both fundamental and advanced SQL techniques. The scripts are designed to simulate a real-world analytics workflow, emphasizing data quality, performance analysis, reporting, and business decision support.
